Setting Up My Other Octopress on Linode

Nov 7th, 2012

Octopress + Rsync + Linode + + Debian +Nginx

I obviously run this blog on Github pages, which is pretty painless and is the easiest blog I have ever set up and administered. But I have a couple other blogs that I wanted to start and I decided to avoid Heroku and host them on my Linode server running Debian Squeeze. I poked around a bit for a good guide on this but had to cobble together the process…so here’s what I did, a cobbled together solution extracted from a bunch of websites but especially from [INCLUDE SITES]:

  1. installed Nginx
  2. added my username to the www-data group: sudo adduser _username_ www-data
  3. change the ownership of the /var/www directory: sudo chown _username_:www-data -R /var/www
  4. change permissions to limit write permissions on the directory to its owner - i.e. you: sudo chmod 0755 -R /var/www
  5. FAIL. One might even say EPIC FAIL. I have failed to follow the various directions I’ve found online in a number of locales. Once I fix the thing I’ll post what I’ve discovered here. I’m surprised I haven’t been able to figure it out as I’ve been using Linux for ten years. But only the buggy but lovable desktop variety - no expereince dealing with servers. Anyway, I’ll figure it out. Most of the things I’ve learned have come through failure and this will be the same.
